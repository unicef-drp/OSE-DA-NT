# Nutrition Data Chart Design Instructions

Use this file as the chart and figure design policy for any agent generating visualizations from nutrition data.

This applies to:
- prevalence (%)
- coverage (%)
- proportions

The goal is to produce charts that are:
- clear
- comparable across countries and time
- aligned with global health reporting standards (e.g. UNICEF, WHO)
- easy to interpret quickly

---

## Core Principle

All charts must answer a clear question.

Bad:
- “Here is some data”

Good:
- “Stunting prevalence is declining but remains above target”

---

## Data Assumptions

Unless explicitly stated otherwise:
- all values are percentages (0–100)
- indicators represent prevalence or coverage
- comparisons are across:
  - time
  - countries
  - regions
  - population groups

---

## Chart Types (Allowed)

### 1. Time Trend (PRIMARY DEFAULT)
Use for change over time.

Type:
- line chart

Rules:
- x-axis: time (years)
- y-axis: percentage (0–100)
- include markers
- max 3 lines

---

### 2. Country Comparison

Type:
- horizontal bar chart

Rules:
- sort descending
- max ~15 countries
- label clearly

---

### 3. Regional Comparison

Type:
- bar or dot plot

---

## Axis Rules

### Y-axis
- always 0–100
- never truncate

### X-axis
- evenly spaced time

---

## Labeling

Include:
- insight title
- axis labels
- % units
- legend if needed

---

## Color Rules

- 1 primary
- 1 accent
- consistent across charts

---

## Targets

- include dashed target line when relevant

---

## Data Density

- avoid overcrowding
- split charts if needed

---

## Annotation

- highlight key insights
- keep short

---

## Formatting

- round to 1 decimal
- use % format

---

## Consistency

- same scale
- same colors
- same ordering

---

## Avoid

- pie charts (many groups)
- 3D charts
- dual axes
- inconsistent scales

---

## Default Behavior

- line = time
- bar = comparison
- always 0–100 scale

---

## Final Rule

Always prioritize clarity over detail.
